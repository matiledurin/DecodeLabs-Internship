01 · Context

What Is Sentiment Analysis?

In plain English: the computer reads a piece of text and decides whether the person writing it feels good or bad about something. No human labels it — the model figures it out from patterns in language.

Why it matters in the real world

Netflix knows which shows viewers love. Amazon flags bad product experiences before they go viral. Banks detect fraud from complaint language. This is the engine behind all of that.

What makes it hard

"Not bad" is positive. "Could have been better" is ambiguous. Sarcasm is a nightmare. Language carries meaning in context, not just words — that's the core challenge.

The dataset

IMDB — 50,000 real movie reviews. Half labelled positive, half negative. Perfectly balanced. This is a benchmark dataset used by researchers worldwide.

The approach

Classic NLP: clean the text, convert words to numbers, train a Naive Bayes classifier, evaluate on unseen data. No deep learning — pure probability and statistics.

02 · Methodology

The Pipeline, Step by Step

Every decision in this project follows a logical sequence. Here's what was built and why each step exists.

01

Data Loading & Exploration

Downloaded 50,000 IMDB reviews via Kaggle. Confirmed perfect class balance (25K positive, 25K negative). Visualised review length distribution — most reviews are 50–300 words, with some outliers.

pandas · matplotlib
02

Text Preprocessing

Stripped HTML tags, removed punctuation, lowercased everything. Removed stopwords (common filler words like "the", "and") — but deliberately kept negation words like "not", "never", "nor" because "not good" and "good" are opposites. Applied lemmatisation: turning "running", "ran", "runs" into their root form "run" so the model treats them the same.

NLTK · regex · lemmatisation
03

TF-IDF Vectorisation

Converted cleaned text into numbers. TF-IDF measures how important a word is in one review relative to the entire dataset. Words that appear everywhere ("film", "movie") get downweighted. Rare, specific words ("breathtaking", "abysmal") get upweighted. Used bigrams (word pairs like "not good") to capture more context. Result: each review becomes a vector of 50,000 numbers.

TF-IDF · bigrams · 50K features
04

Model Training

Split data 80/20 — 40,000 reviews to train on, 10,000 to test with. Trained Multinomial Naive Bayes: a fast, effective algorithm that uses probability ("given this word appears, how likely is this review positive?"). Also trained Complement Naive Bayes as a comparison — it's designed to handle text classification better in some cases.

MultinomialNB · ComplementNB · stratified split
05

Evaluation & Comparison

Measured accuracy, precision, recall, and F1-score on the unseen test set. Generated a confusion matrix to visualise where the model is right and where it makes mistakes. Built a live prediction function to test on custom text.

classification_report · confusion matrix · predict_proba

What's actually happening under the hood

The model has learned which words appear most in positive reviews (brilliant, masterpiece, loved, moving) versus negative ones (awful, boring, waste, terrible). It calculates: "given the words in this sentence, what's the probability it's positive vs negative?" — and picks the higher one. It also gives you a confidence score.

05 · Reflection

What This Project Actually Teaches

The accuracy number is the least interesting part. Here's what the process reveals about how language and machines interact.

🎯

Negation is a precision decision, not a default

Most NLP tutorials remove all stopwords blindly. This project deliberately kept "not", "never", "nor" — because "not good" is the opposite of "good." That's a conscious engineering choice that improves accuracy.

🔢

You can't feed words to a machine — only numbers

TF-IDF transforms language into a mathematical representation. It weights rare, specific words higher than common ones. This is how the model "understands" that "breathtaking" means more than "the".

⚖️

Simple models are underrated

Naive Bayes assumes all words are independent (which isn't true). Yet it achieves ~87% accuracy on 50,000 real reviews. Sometimes the simpler model with good preprocessing beats the complex one with bad data.

🔍

Accuracy alone is misleading

A model that guesses "positive" every single time would get 50% accuracy on this dataset. That's why precision, recall, and F1-score matter — and why the confusion matrix shows the full picture.

🌍

The real-world applications are immediate

Brand monitoring, customer support triage, review analysis, social media tracking — this same pipeline, scaled up, powers billion-rand products. The concepts here are the foundation.

06 · Tools Used

The Technical Stack

Every tool had a specific reason to be here. This wasn't a random selection — each library solves a distinct problem in the pipeline.

Data & Numbers

pandas numpy

Load, inspect, and manipulate the dataset. pandas handles the dataframe; numpy handles the matrix math under TF-IDF.

Language Processing

NLTK regex

Tokenisation, stopword removal, part-of-speech tagging, and lemmatisation. NLTK is the standard NLP library in Python.

Feature Engineering

TfidfVectorizer

Converts cleaned text into 50,000-dimensional numerical vectors. The bigram parameter adds word-pair context.

Machine Learning

MultinomialNB ComplementNB sklearn.metrics

Two Naive Bayes variants trained and compared. sklearn.metrics provides the full evaluation suite.

Visualisation

matplotlib seaborn

Sentiment distribution, review length histogram, TF-IDF feature chart, confusion matrix, and confidence distribution.

Data Source

kagglehub IMDB 50K Dataset

Industry-standard benchmark dataset. 50,000 real user reviews, perfectly balanced between positive and negative.
